## 2. 기능 상세 정리 (Functional Requirements)

### 2.1. 디지털 옷장 자동 등록 시스템 (Digital Closet)
* **[FR-1.1] 옷 사진 업로드 및 자동 배경 제거 (Background Removal)**
  * **기능:** 사용자가 바닥에 옷을 놓고 촬영하거나 입고 찍은 사진을 업로드하면, 의류 영역만 정확히 세그멘테이션하여 배경을 투명하게 처리(누끼).
  * **로직:** Segment Anything Model(SAM) 또는 Remove.bg API를 거쳐 옷 단일 이미지(PNG) 추출 및 저장.
* **[FR-1.2] AI 자동 의류 태깅 (Auto-Tagging)**
  * **기능:** 업로드된 의류 이미지를 분석하여 메타데이터 자동 추출 및 제안.
  * **태그 정보:** 카테고리(상의-셔츠/니트/티셔츠 등, 하의-슬랙스/데님 등, 아우터), 색상(RGB 매핑 기반 대표 색상명), 패턴(무지, 스트라이프, 체크 등), 계절감(봄/여름/가을/겨울).
  * **예외 처리:** AI 태깅 결과가 오분류되었을 경우를 대비해, 사용자가 직접 드롭다운으로 태그를 수정할 수 있는 편집 폼 제공.

### 2.2. 체형 분석 및 기상 환경 수집 (Context Engine)
* **[FR-2.1] 초개인화 체형 프로필 설정**
  * **기능:** 최초 온보딩 시 사용자의 신체 스펙 입력 지원.
  * **필수 입력 스펙:** 성별, 키, 몸무게.
  * **선택형 체형 특징:** 상체 발달형(역삼각형), 하체 발달형(삼각형), 허리 강조형(모래시계형), 일자형 등 시각화 패널 선택 제공.
* **[FR-2.2] 실시간 위치 기반 날씨 API 연동**
  * **기능:** 기상청 단기예보 API 연동을 통한 사용자 현재 위치의 시간별 기온, 강수 확률, 미세먼지 데이터 수집.
  * **의류 추천 변수 변환:** 기온에 따라 최적의 '클로즈 레이어링 지수(Clothing Insulation, clo)'를 계산하여 코디 매칭 엔진의 입력 변수로 전달.

### 2.3. 하이브리드 코디 매칭 및 스타일 추천 엔진 (Styling Agent)
* **[FR-3.1] 오늘 날씨 & TPO 맞춤형 코디 제안**
  * **기능:** 메인 화면 접속 시 오늘 날씨 정보와 매칭되는 옷장 속 베스트 착장 조합(상의+하의+아우터) 3가지 제안.
  * **TPO 옵션 필터:** '출근(비즈니스 캐주얼)', '데이트(러블리/댄디)', '편안한 일상(캐주얼/원마일웨어)' 선택 필터 제공.
  * **추천 알고리즘:**
    1. Fashion-CLIP 임베딩을 이용해 보관된 의류 벡터 간 어울림(색상 조화도, 스타일 일치성) 계산.
    2. 사용자 체형 조건 가중치 적용 (예: 하체 발달형인 경우 와이드 핏 바지 우선 매칭).
    3. LLM 프롬프트에 조합 정보를 입력하여 코디 추천 사유(Reasoning)를 2줄 요약 텍스트로 생성.
* **[FR-3.2] 부족한 아이템 스마트 쇼핑 추천**
  * **기능:** 옷장 속 옷으로 최상의 코디를 제안하되, 특정 아이템(예: 화이트 이너 티셔츠, 브라운 벨트)이 부족할 경우 해당 코디를 완성하기 위해 필요한 보완 아이템 추천 및 외부 쇼핑몰 구매 링크 노출.

## 3. UX Flow (User Journey Map)

### 3.1. 옷 등록 플로우 (Upload Flow)
```
[옷장 홈 화면]
    └─ (Tab) 오른쪽 아래 [+] 버튼 클릭
         └─ [카메라 구동 및 업로드 선택]
              ├─ 직접 촬영 혹은 앨범에서 선택
              ├─ [AI 이미지 분석 중 로딩 스냅샷] (배경 제거 및 특징 추출 동시 진행)
              └─ [태그 확인 및 수정 페이지]
                   ├─ 누끼 따진 깔끔한 의류 이미지 미리보기
                   ├─ 카테고리, 색상, 패턴, 계절 자동 세팅 데이터 확인
                   └─ (Tab) [내 옷장에 저장] 버튼 클릭 -> 완료 팝업 및 옷장 목록 리다이렉트
```

```
[메인 대시보드 화면]
    ├─ 상단: 오늘 현재 기온 (예: "현재 22°C, 화창함") 및 TPO 슬라이더 (출근 / 일상 / 데이트)
    ├─ 중앙: [AI 베스트 스타일링 코디 카드] 롤링 컴포넌트 (상의, 하의, 아우터가 배치된 레이아웃)

    │     └─ 카드 하단: "하체 라인을 자연스럽게 커버해 주는 세미와이드 슬랙스와 단정한 니트 조합입니다." (AI 추천 사유)
    └─ 하단 영역: "이 코디에 어울리는 추천 아이템" (부족한 보완재 아이템 무신사 등 외부 구매 링크 노출)
```

## 4. 기술 스택 (Tech Stack)

비전 분석 처리와 유연한 코디 알고리즘, 대용량 이미지 렌더링에 적합한 검증된 테크 스택입니다.

```
┌──────────────────────────────────────────────────────────────────┐
│                           Client App                             │
│ ┌──────────────────────────────────┐ ┌─────────────────────────┐ │
│ │          React Native            │ │   Zustand & React Query │ │
│ │ - 네이티브 카메라 롤 & 이미지 캐싱 │ │ - 전역 상태 및 API 상태  │ │
│ └──────────────────┬───────────────┘ └────────────┬────────────┘ │
└────────────────────┼──────────────────────────────┼──────────────┘
                     │ REST API (JSON)              │ Image Upload
┌────────────────────▼──────────────────────────────▼──────────────┐
│                        Backend & AI Platform                     │
│ ┌──────────────────────────────────────────────────────────────┐ │
│ │                       FastAPI (Python)                       │ │
│ │  - 비동기 이미지 업로드 처리 및 메타 태깅 라우터              │ │
│ │  - 외부 날씨(기상청) API 캐싱 및 TPO 규칙 엔진 연동             │ │
│ └───────┬───────────────┬───────────────────────────────┬──────┘ │
└─────────┼───────────────┼───────────────────────────────┼────────┘
          │               │                               │
┌─────────▼────────┐ ┌────▼─────────────┐          ┌──────▼────────┐
│     AI Model     │ │    Database      │          │ Cloud Storage │
│ - SAM/U2Net OCR  │ │ - PostgreSQL     │          │ - AWS S3      │
│   (누끼 처리 API) │ │   (유저/메타데이터)│          │   (누끼 처리  │
│ - Fashion-CLIP   │ │ - Qdrant         │          │    의류 이미지│
│   (의류 매칭 임베딩│ │   (의류 벡터 DB) │          │    영구 보관) │
│ - OpenAI GPT-4o  │ └──────────────────┘          └───────────────┘
└──────────────────┘
```

* **Frontend:**
  * **React Native:** iOS와 Android 동시 릴리즈 및 모바일 하드웨어 카메라/앨범 API 액세스 최적화.
  * **Zustand:** 사용자의 실시간 필터(TPO, 날씨 선호도) 및 옷 세션 정보 상태 관리.
* **Backend:**
  * **FastAPI:** Python 이미지 프로세싱 라이브러리(Pillow, OpenCV) 및 AI Inference 라이브러리와 최상의 연동 속도 보장.
* **Database & Storage:**
  * **PostgreSQL:** 유저 신체 프로필 정보, 옷의 메타데이터(카테고리, 태그, 구매 정보 등) 관계형 데이터 저장.
  * **Qdrant (Vector DB):** 등록된 옷의 패션 임베딩 벡터값들을 보관하고, 유사 스타일 및 보완 아이템 검색의 초고속 유사도 계산($L_2$ Distance 또는 Cosine Similarity)에 활용.
  * **AWS S3 & CloudFront:** 누끼 가공이 완료된 고화질 의류 PNG 파일을 모바일에 딜레이 없이 렌더링하기 위한 CDN 파이프라인.
* **AI Engine:**
  * **U2Net / SAM (Segment Anything Model):** 정밀한 의류 누끼 추출을 위한 Edge-detection 기반 딥러닝 백엔드 모듈.
  * **Fashion-CLIP:** 패션 도메인에 파인튜닝된 OpenAI CLIP 기반 멀티모달 모델. 업로드 이미지에서 무드 및 어울림 특질 벡터를 산출해 매칭 알고리즘의 기초 데이터로 활용.
  * **OpenAI GPT-4o-mini:** 텍스트 생성 전용. 매칭된 코드와 날씨 정보를 바탕으로 사용자 친화적인 대화형 설명 텍스트 실시간 큐레이팅.

## 5. MVP 로드맵 (Milestone)

* **Sprint 1 (W1-W2): 옷장 이미지 처리 및 파이프라인 구축**
  * SAM/U2Net API를 이식한 누끼 추출 마이크로서비스 테스트 및 완료.
  * 옷장 업로드 프론트엔드 퍼블리싱 및 AWS S3 저장 연동 확인.
  * AI 자동 태깅용 Fashion-CLIP 카테고리 매핑 레이블러 구현.
* **Sprint 2 (W3-W4): 체형-날씨 연계 추천 코어 엔진 개발**
  * 기상청 API 연동을 통한 지역 날씨 정보 파싱 및 DB 내부 캐싱 스케줄러 구현.
  * 사용자 신체 정보 기반의 필터링 규칙 엔진 코딩.
  * Qdrant Vector DB를 도입하여 저장된 상하의 벡터 유사도 연산을 통한 코디 조합 추천 알고리즘 1차 완성.
* **Sprint 3 (W5): 서비스 연동 및 최적화**
  * 메인 코디 추천 대시보드 UI 연동 및 부드러운 카드 스와이프 인터랙션 구현.
  * 생성된 추천 사유 문구(GPT-4o-mini API 연동) 속도 최적화 및 텍스트 캐싱 적용.
  * 쇼핑몰 파트너십 API를 통한 추천 상품 연계 마크업 작업.
* **Sprint 4 (W6): 배포 전 QA 및 FGT**
  * 2030 핵심 타깃 오디언스 대상 클로즈베타(FGT) 테스트 및 피드백 취합.
  * 부적절한 이미지 업로드 시 예외 처리 및 옷 등록 실패 복구 메커니즘 튜닝 후 스토어 정식 배포.

## 6. 리스크 및 대응 전략 (Risk & Mitigation)

### Risk 1. 저화질 혹은 배경이 너무 복잡한 옷 사진의 누끼 오작동 및 태깅 오류
* **대응 전략:** * 이미지 업로드 화면 진입 시 가이드라인 가이드 오버레이(예: "단색 바닥에 옷이 한눈에 보이도록 정돈해 주세요")를 직관적으로 제공합니다.
  * 자동 배경 분할 스코어가 특정 임계값(예: IoU 0.6) 미만인 경우, 사용자에게 수동 크롭 도구(Crop Tool)를 노출하여 직접 영역을 잡아줄 수 있는 대체 UX를 설계합니다.

### Risk 2. 옷장에 등록된 의류 개수가 너무 적어(예: 3벌) 추천할 수 있는 코디 조합이 없는 경우
* **대응 전략:** * 옷장의 옷 개수가 5벌 미만인 초반 가입 유저에게는 사전에 구성된 '기본 스타일링 템플릿(가상의 기본 의류 조합)'을 먼저 제시합니다.
  * "현재 가지고 계신 청바지에 이 기본 화이트 셔츠 하나만 더하면 완벽한 남친룩이 됩니다!"와 같은 안내와 함께 부족한 기본 아이템 쇼핑 카드로 자연스럽게 연계하여 초기 유저 이탈을 막고 구매 전환율을 높입니다.

### Risk 3. 날씨 변화 및 TPO 필터 다양화에 따른 추천 오버헤드 및 API 비용 급증
* **대응 전략:**
  * 날씨 정보는 유저가 앱을 켤 때마다 호출하지 않고 Redis 캐시에 사용자 리전(Region) 단위로 최소 1시간 단위 캐싱을 적용합니다.
  * GPT API 호출 비용 제어를 위해, 코디 추천 이유 텍스트 템플릿의 프롬프트를 완전 자유도 방식이 아닌, 일정한 템플릿 기반 채우기 구조(Few-Shot Prompting 및 Structured Outputs)로 한정하여 입력/출력 토큰 크기를 극소화합니다.
